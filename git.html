gitignore

A .gitignore file is used in Git to specify files and directories that should be ignored and not tracked by version control. This is useful for excluding files generated by the build process, temporary files, configuration files, and other files that shouldn't be part of the repository. Here's a basic example of a .gitignore file:

# Ignore build artifacts
build/
dist/

# Ignore configuration files
*.config

# Ignore temporary files
*.tmp
*.log

# Ignore files generated by IDEs
.idea/
.vscode/
