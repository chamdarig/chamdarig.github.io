cp command

The cp command in Linux is used to copy files and directories from one location to another. Here is the basic syntax:

cp [options] source destination
  • source: Specifies the file or directory to be copied.
  • destination: Specifies the destination where the file or directory will be copied.

    Here are some common options you can use with the cp command:

    • -r or -R: Copy directories recursively.
    • -i: Prompt before overwriting files.
    • -u: Copy only when the source file is newer than the destination file or when the destination file is missing.
    • -a or --archive: Copy files and directories recursively and preserve as much as possible, including ownership, permissions, and timestamps.
    • -v or --verbose: Enable verbose mode, showing the files being copied.

    Examples:

    1. Copy a file to another location:

      cp file.txt /path/to/destination/
      2. cp -r source_directory/ /path/to/destination/

    2. Copy with confirmation before overwriting:

      cp -i file.txt /path/to/destination/

    3. Copy with verbose output:

      cp -v file1.txt file2.txt /path/to/destination/

    Always be cautious when using the cp command, especially with options that can overwrite existing files. Double-check your commands to avoid unintentional data loss.

    cp file1.txt file2.txt /path/to/destination/
